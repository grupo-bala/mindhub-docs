## 1. Introdução

### 1.1. Objetivo

Este documento tem como finalidade especificar o processo utilizado para o desenvolvimento do sistema MindHub.

### 1.2. Visão Geral

O processo foi modelado de acordo com as necessidaddes de entregas da disciplina de Projeto Integrado em Engenharia de Software III, sendo dividido em 3 Status Reports, sendo baseado nas metodologias XP e Scrum. Cada fase do projeto conta com atividades de planejamento, modelagem e documentação, e em seguida uma sequência de iterações de desenvolvimento.

## 2. Descrição do processo

Seguindo os modelos ágeis de desenvolvimento, o processo é iterativo e baseado em sprints de duração fixa de 1 semana, onde histórias de usuários são alocadas para serem desenvolvidas. Antes da implementação é feita uma análise do feedback da entrega anterior, e uma preparação para melhorias de acordo com esse feedback.

As histórias de usuários são agrupadas em épicos que descrevem as funcionalidades de alto nível da aplicação. O cronograma de implementação é gerenciado pelo Jira, de acordo com as datas de entrega e o tamanho de cada épico.
