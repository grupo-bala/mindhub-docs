## Tecnologias

### Aplicativo android

Para as tecnologias do aplicativo Android escolhemos as seguintes tecnologias:

- [Kotlin](https://kotlinlang.org/): devido a sua recomendação pela Google
- [Android jetpack](https://developer.android.com/jetpack): por facilitar o desenvolvimento
- [Material Design 3](https://m3.material.io/): para servir como biblioteca de componentes visuais prontos e testados
- [Google Maps API](https://mapsplatform.google.com/?hl=pt-br): para fornecer serviços de geolocalização

Ademais, utilizaremos o [Android Studio](https://developer.android.com/studio) como ambiente de desenvolvimento.

### Servidor

Para as tecnologias do servidor escolhemos as seguintes tecnologias:

- [TypeScript](https://www.typescriptlang.org/): devido a tipagem estática que facilitará a produção de código ao longo do projeto
- [NestJS](https://nestjs.com/): pela facilidade de desenvolver usando o framework e pela integração com o swagger
- [TypeORM](https://typeorm.io/): integração com TypeScript e Postgresql
- [Vitest](https://vitest.dev/): devido a sua alta performance e integração com o TypeScript
- [Eslint](https://eslint.org/): consistência de estilo e boas práticas

### Banco de dados

Para a tecnologia do banco de dados optamos por escolher o [Postgresql](https://www.postgresql.org/) como tecnologia para o banco de dados devido a nossa familiaridade com ele, por implementar um modelo relacional no banco de dados e por ter integração com a biblioteca de ORM escolhida para o servidor.

## Ferramentas

### Github

Decidimos utilizar o [Github](https://github.com/) para armazenar o código tanto do front-end quanto do back-end e também iremos utilizar a funcionalidade do [Github Actions](https://github.com/features/actions) para automação de testes, checagem de linter e deploy automático.

## Hospedagem

### Servidor

Escolhemos o [Fly](https://fly.io/) por ser uma ferramenta de hospedagem gratuita que possui máquinas para hospedagem no Brasil e possui uma configuração fácil e intuitiva.
